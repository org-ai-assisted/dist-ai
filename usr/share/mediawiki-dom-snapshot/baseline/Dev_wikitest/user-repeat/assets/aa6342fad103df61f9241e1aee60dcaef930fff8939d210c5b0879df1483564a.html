CSS Extension Fork

[edit]
  • These test cases cover all functionality and error conditions of the {{#css:}} parser function.
  • The results are checked via javascript. Detailed results are rendered into the "detailed results" area below
  • Tests are structured like this : PASSED/FAILED | Test ID | Description | Input | Expected Result

A. Basic Functionality

  1. Wait... | A1 | Empty input | {{#css:}} | No output, no <style> tag
  2. Wait... | A2 | Simple inline CSS | {{#css:#made-up-id1 { color: red; }}} | <style type="text/css">made-up-id1{color:red}</style> added to page head
  3. Wait... | A3 | Multiple inline rules and leading / trailing whitespaces | {{#css:#made-up-id2 { font-size: 2em; } #made-up-id3 { font-size: 1.5em; }}} | Both rules minified in a single <style> tag and whitespace trimmed, CSS parsed normally

B. File-Based CSS

  1. Wait... | B1 | Local file path (absolute) | {{#css:/src-copy/CssExtensionForkTests.css}} | <link rel="stylesheet" href=".../mw-autogen/src-copy/CssExtensionForkTests.css?css-extension=1&hsversion_from_server_replacement_unixtime=1778923154&hsversion_from_server_replacement_unixtime=1778583349&hsversion_from_server_replacement_unixtime=1778583349&hsversion_from_server_replacement_unixtime=1778583349&hsversion_from_server_replacement_unixtime=1778583349&hsversion_from_server_replacement_unixtime=1778583349&hsversion_from_server_replacement_unixtime=1778583349&hsversion_from_server_replacement_unixtime=1778583349&hsversion_from_server_replacement_unixtime=1778583349&hsversion_from_server_replacement_unixtime=..." />
  2. Wait... | B2 | Local file path (relative) | {{#css:/resources/test-unreachable-expected-failure.css}} | Relative path resolved against $wgCSSPath
  3. Wait... | B3 | File path with path traversal attempt | {{#css:/src-copy/../../../etc/passwd}} | After normalization, blocked with ""
  4. Wait... | B4 | URL-encoded path traversal (security test) | {{#css:/src-copy/%2e%2e/%2e%2e/test-unallowed-traversal-to-server-secrets-failure-expected.txt}} | Should be BLOCKED by extension. If it returns a valid <link> tag, the sanitizer is vulnerable to URL-encoded traversal.
  5. Wait... | B5 | File path with backslash traversal | {{#css:/src-copy\..\..\etc\passwd}} | Backslashes converted to forward slashes, then blocked

C. Article-Based CSS (Whitelist)

  1. Wait... | C1 | Article in whitelisted namespace | {{#css:MediaWiki:Common.css}} | <link rel="stylesheet" href=".../MediaWiki:Common.css?action=raw&ctype=text/css&css-extension=1" />
  2. Wait... | C2 | Article in non-whitelisted namespace | {{#css:User:Patrick/myskin.css}} | HTML comment error: "Only namespaces [...] allowed. You use: 2 (namespace id)"
  3. Wait... | C3 | Non-existing article | {{#css:MediaWiki:NonExistent.css}} | Article does not exist → treated as inline CSS → sanitizer fails with /* css-sanitizer failed to parse CSS */
  4. Wait... | C4 | Article without namespace prefix | {{#css:ExactlyThirtyOneCharactersLong.css}} | No namespace prefix → not recognized as article → treated as inline CSS → sanitizer fails with /* css-sanitizer failed to parse CSS */

D. CSS Sanitizer

  1. Wait... | D1 | Disallowed property | {{#css:body { behavior: url(script.htc); }}} | Property removed by sanitizer
  2. Wait... | D2 | @import rule (BLOCKED) | {{#css:@import url('https://evil.com/bad.css');}} | @import removed by sanitizer (not in allowed list)
  3. Wait... | D3 | Malformed CSS (parse error) | {{#css:body { color: red; {} }} | Output: /* css-sanitizer failed to parse CSS */
  4. Wait... | D4 | CSS that fails sanitization | {{#css:body { -evil-vendor-property: bad-value; }} | Property silently removed by sanitizer, output CSS without the disallowed property
  5. Wait... | D5 | Closing style tag injection | {{#css:body { content: "</style><script>alert('xss')</script>"; } }} | Output: /* Closing style tag found in resulting CSS */
  6. Wait... | D6 | U+007F control character | {{#css:body { content: "\7f "; } }} | U+007F escaped to \7f in CSS output

E. Edge Cases

  1. Wait... | E1 | CSS with comments | {{#css:/* comment */ body { color: red; }} | Comment stripped, valid CSS remains
  2. Wait... | E2 | CSS custom properties | {{#css::root { --main-color: #ff0000; } }} | Custom properties not supported by older sanitizer → /* css-sanitizer failed to sanitize CSS */
  3. Wait... | E3 | Multiple CSS calls on one page | Two separate {{#css:...}} calls (this is already given by the other tests) | Both processed independently, multiple <style> or <link> tags
